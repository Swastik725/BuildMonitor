# 🤝 CONTRIBUTING.md

Version: 2.0 (current build — solo dev, build sprint)

---

## Branch Strategy (current build)

- `main` — always deployable
- `feature/*` — one branch per roadmap item from `ROADMAP.md`

(`develop` and `hotfix/*` are dropped for current build — unnecessary ceremony for a solo build build.)

---

## Commit Convention

feat: / fix: / docs: / refactor: / test: / perf: / chore:

---

## Rules (current build)

- No direct push to `main` once the app is live (Day 10 onward) — small feature branches instead
- Update `ROADMAP.md` checkboxes as items complete, so progress stays honest against the build
  clock
